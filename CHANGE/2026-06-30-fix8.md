## 问题列表

### P1: 引用面板图片不显示
- **现象**: RAG 回答的"查看引用"面板中只有文档名和相似度，没有文档图片缩略图
- **原因**: PDF 是在旧版解析器（不支持图片提取）之前上传的。ChromaDB 中存储的 chunk 文本不包含 `[图片文件] {path}` 标记，`_extract_image_paths()` 返回空数组
- **涉及文件**:
  - `backend/app/services/parser.py` — 已支持图片提取（pypdf page.images）
  - `backend/app/services/rag.py` — 已支持 image_paths 提取和返回
  - `backend/app/api/chat.py` — 已在 sources 中返回 images 数组
  - `src/api/search.ts` — 已映射 s.images 到 meta.images
  - `src/views/search/Pc.vue` — 已渲染 h.images 缩略图（行399-403）
- **修复方案**: **不需要改代码**，用户需要：
  1. 删除知识库中的旧 PDF
  2. 重新上传 → 触发新版 parser 图片提取流程
  3. ChromaDB 重新入库，chunk 包含 `[图片文件]` 标记
- **验证方法**: 搜索后展开引用，应看到 w-28 h-20 的图片缩略图

### P2: 搜索框输入文字切换页面后丢失
- **现象**: 在搜索框输入文字后，切到其他页面再回来，输入文字消失
- **根因**: Pc.vue `onMounted()` 的分支逻辑有缺陷：
  ```js
  if (query.q) → auto-send
  else if (activeSession) → restore session   // ← 走这里就跳过了草稿恢复！
  else → restore draft                        // ← 只有没会话时才走这里
  ```
  用户发过至少一条消息后就永远有 activeSession，草稿分支永远不会执行
- **修改文件**: `src/views/search/Pc.vue`
- **修改内容**: 将草稿恢复逻辑从 else 分支中**提取出来**，无论是否有活动会话都执行：
  ```js
  onMounted(() => {
    if (input.value) { onSend() }
    else if (activeSession) { restoreSession() }
    // 始终恢复草稿（独立于上面的分支）
    const d = store.draft
    if (!input.value && d.question) input.value = d.question
  })
  ```

### P3: DB_URL 路径解析报错（SQLAlchemy 无法解析）
- **现象**: 启动 uvicorn 报错 `Could not parse SQLAlchemy URL from given URL string`
- **根因**: config.py 将相对路径 DB_URL (`sqlite:///./loongchip.db`) 转为 Windows 绝对路径后包含反斜杠 (`sqlite:///F:\software\...`)，SQLAlchemy URL 解析器不认反斜杠
- **修改文件**: `backend/app/core/config.py`
- **修改内容**:
  1. 路径基准目录修正：从 `dirname(dirname(abspath(__file__)))` 改为 `dirname(dirname(dirname(abspath(__file__))))` （config.py 在 app/core/ 下，需往上3层到 backend/）
  2. DB_URL 单独处理：用 `.replace("\\", "/")` 确保正斜杠

## 数据流说明

```
用户上传 PDF
  ↓ parser.py: pypdf PdfReader → page.images → 保存到 uploads/extracted_images/{docname}_{hash}/
  ↓ 同时写入文本标记: [图片文件] F:\...\uploads\extracted_images\...\page-001-image-01.png
  ↓
rag.py: split_text() 切片 → _extract_image_paths() 从 chunk 提取 [图片文件] 行
  ↓ ingest_document(): metas["image_paths"] = "|".join(paths) 存入 ChromaDB
  ↓
用户提问 → rag.py search() 返回 hits["image_paths"]
  ↓ chat.py query(): _image_items(paths) → [{url: "/api/kb/image/page-001-image-01.png", name: "..."}]
  ↓ search.ts: s.images → meta.images
  ↓ Pc.vue: <img :src="img.url"> 渲染缩略图
  ↓ 用户点击 → /api/kb/image/{name} → kb.py FileResponse 返回图片文件
```
