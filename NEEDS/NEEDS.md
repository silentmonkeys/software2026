# 设备检修知识检索与作业系统 · 前端设计提示词（完整版）

## 一、项目背景

请设计一套 **B/S 架构** 的 PC Web 前端页面，用于**"基于多模态大模型技术的设备检修知识检索与作业系统"**。

**业务场景**：钢铁、汽车制造等行业的产线设备检修现场。一线检修员需要在车间快速检索检修手册、诊断故障、获取标准化作业流程；知识管理员需要审核并沉淀一线提交的案例；系统管理员需要管理用户、知识库与流程模板。

**核心用户角色**：

1. **一线检修员**（最高频）：拍照+文字+设备型号检索；跟着 SOP 流程做检修；遇到新问题上传案例
2. **知识审核员**：审核一线提交的案例是否入库
3. **系统管理员**：管理设备型号库、流程模板、用户与权限
4. **访客/新用户**：只读检索，注册后可使用全部功能

**部署硬约束（务必体现在 UI 上）**：

- 必须运行在 **LoongArch 架构 + 银河麒麟高级服务器操作系统 V11/V10** 上
- 分辨率主要在 **1920×1080 / 2K 工业大屏**上使用
- 浏览器推荐 **Chrome 100+ / 麒麟自带浏览器**
- **同时需要支持车间现场的移动端浏览器访问**（手机/平板），详见第八节

## 二、设计风格

**整体调性**：**专业工业感 × 现代科技感 × 现场可用性**

不要做成冷冰冰的"企业后台"，也不要太花哨。要让车间老师傅 5 分钟内能上手，又要让甲方领导觉得"这是个有 AI 含量"的产品。

| 维度       | 要求                                                                                            |
| ---------- | ----------------------------------------------------------------------------------------------- |
| 视觉关键词 | 工业、机械、精密、清晰、AI 智能、现场感                                                         |
| 色彩       | 深空蓝 + 机械橙 + 中性灰白（详见配色）                                                          |
| 风格参考   | 工业仪表盘（HMI）、特斯拉车主端 UI、Notion 干净排版、Cursor IDE 的深色侧栏                      |
| 圆角       | 中等圆角 8–12px，避免过度圆润显得不专业；按钮 6–8px                                             |
| 阴影       | 克制，多用 1px 边框 + 极淡阴影，避免厚重卡片感                                                  |
| 字体       | 优先无衬线：思源黑体 / 苹方 / Inter；数字与设备型号用等宽字体（JetBrains Mono / DIN）增强工业感 |
| 图标       | 线性图标为主（Lucide / Tabler Icons），关键按钮可点缀双色图标                                   |
| 间距       | 8 的倍数节奏（8/16/24/32），保持工业图纸般的规整感                                              |

**配色方案（直接使用）**：

```

成功             Success   #2E7D32

中性背景         BG        #F5F7FA

正文             Text      #1F2937
次要文字         Text-2    #6B7280
反白文字         OnDark    #E5E7EB
```

**深色模式**（加分项）：顶栏/侧栏深色 `#0B2545`，工作区浅色 `#F5F7FA`，车间夜间或长时间使用更护眼。

## 三、页面清单（必交付）

请按以下页面结构设计，每个页面给出 **线框图描述 + 关键组件 + 交互细节**。可以输出 HTML/React/Vue 静态稿，或单页 Figma 结构化描述。

### 1. 登录页 `/login`

- 居中卡片，宽度 420px
- 背景：抽象的工业齿轮 + 神经网络线条的 SVG 渐变（蓝橙双色）
- 表单：账号、密码、记住我、登录按钮（机械橙）
- 下方：企业 SSO 入口、忘记密码、注册链接
- 顶部 Logo：左侧"检修助手" + 右侧"LoongArch | 银河麒麟"小徽标，彰显信创身份

### 2. 主控台/首页 `/dashboard`

- **顶部栏（高 56px）**：Logo + 全局搜索框（按 ⌘K 唤起命令面板）+ 通知铃铛 + 用户头像
- **左侧栏（宽 220px）**：导航菜单（图标+文字）
  - 工作台（默认）
  - 多模态检索
  - 作业指引
  - 知识库
  - 案例审核（仅审核员可见，带数量徽标）
  - 知识图谱
  - 系统管理（仅管理员）
- **工作区**采用 12 栏栅格，包含 4 个区块：
  1. 欢迎条 + 今日待办（3 条）
  2. 快捷入口卡片网格（4 张：拍照诊断、语音输入、SOP 模板、我提交的案例）
  3. 最近检索记录列表（5 条，可点击重跑）
  4. 知识库统计卡片（总案例数、本周新增、Top 5 故障类型柱状图）

### 3. 多模态检索页 `/search`（核心页）

**布局**：左侧筛选 + 右侧结果（4:8 分栏）

**左侧筛选栏**：

- 设备类型树（按"行业→车间→设备"三级）
- 检修等级（一级/二级/三级）多选
- 时间范围、来源类型（手册/案例/图谱）
- 命中关键词高亮设置

**主检索区**（核心）：

- 顶部多模态输入条：
  - 文本框（占位："描述故障现象，如：电机异响、温度异常…"）
  - 拖拽/粘贴图片区（虚线边框，hover 时变橙，拖入时变实线 + 缩略图）
  - 设备型号下拉（可搜索、可手动输入）
  - "AI 智能检索"主按钮（橙底白字，带 ✨ 闪光动效）
- 检索模式切换：精准 / 智能（推荐）/ 探索
- 结果区：
  - 顶部摘要：AI 生成的"一句话诊断 + 可能原因 Top3"（带置信度进度条）
  - Tab 切换：全部 / 手册片段 / 历史案例 / 图谱节点
  - 卡片列表：每张卡片显示
    - 标题 + 类型徽标（手册📖 / 案例📋 / 图谱🔗）
    - 相似度评分（橙色进度条 + 百分比）
    - 关键片段（命中关键词高亮）
    - 来源（手册名 + 页码 / 案例提交人 / 图谱实体）
    - 操作：查看详情、收藏、相关推荐
  - 分页/无限滚动
- 右下角悬浮"AI 助手"按钮，唤起对话侧栏，可追问

### 4. 检索详情抽屉/页

- 打开时为右侧抽屉，宽 720px
- 顶部：标题 + 来源元信息 + 关闭按钮
- 中部：内容渲染
  - 手册类：PDF 预览 + 命中段落高亮 + 同段落图片放大
  - 案例类：完整描述 + 图片轮播 + 提交人/时间 + 👍👎 反馈
  - 图谱类：实体卡片 + 关系可视化（局部图）
- 底部固定操作栏：收藏、复制、相关推荐、"以此为上下文继续提问"

### 5. 标准化作业指引页 `/workflow`

- 顶部：设备型号 + 检修等级 + 流程名称选择器
- 流程时间线（左侧步骤序号，右侧步骤卡片）：
  - 当前步骤高亮（左侧橙边 + 背景浅橙）
  - 每步包含：步骤名、预计耗时、操作要点、合规校验点
  - 危险步骤显示 ⚠️ 警告条
  - 必填确认复选框（必须勾选才能"下一步"）
- 顶部进度条：当前 / 总步骤 + 预计剩余时间
- 右侧浮动：当前步骤的工具/物料清单、关联手册链接
- 完成后：弹出总结页，可上传现场照片、填写异常、生成检修报告

### 6. 知识上传/案例提交页 `/knowledge/upload`

- 多步表单（步骤条）：
  1. 基本信息：标题、设备型号、故障类型（自动联想）、检修等级、发生时间
  2. 故障描述：富文本编辑器（支持图片粘贴、表格、代码块）
  3. 处理过程：步骤化记录（可从 SOP 模板导入）
  4. 结果与反思：处理结果、耗时、经验总结
  5. 附件：图片/PDF/视频（拖拽上传，显示进度）
- 右侧实时预览面板（PC 端宽屏才显示）
- 底部：保存草稿 / 提交审核（带"提交前 AI 预检"提示）

### 7. 案例审核工作台 `/audit`

- 顶部 Tab：待审（默认）/ 已通过 / 已驳回 / 我的审核
- 列表视图：表格，列：标题、提交人、设备型号、提交时间、状态、AI 预审结果、操作
- 详情：左侧原文预览，右侧审核面板（通过/驳回 + 批注 + 一键抽取关键信息入图谱）
- 批量操作：勾选多条，批量通过/驳回

### 8. 知识图谱可视化页 `/kg`

- 全屏画布
- 中心搜索框 + 实体类型筛选（设备/部件/故障/处理方法）
- 图谱渲染：力导向图，节点大小=关联案例数，颜色=实体类型
- 节点 hover：弹出卡片显示属性 + 关联案例 Top3
- 节点点击：右侧抽屉显示详情
- 工具栏：放大缩小、布局切换、导出图片、生成子图

### 9. 历史与收藏 `/history`

- 时间轴样式 + 筛选器
- 检索历史、收藏案例、收藏手册、对话历史 分 Tab
- 支持批量删除、导出

### 10. 用户中心 `/profile`

- 左：头像、昵称、角色徽标、所属车间
- 右：Tab 切换——基本信息 / 安全设置 / 通知偏好 / 我的贡献（提交案例数、被采纳数、积分）

### 11. 系统管理 `/admin`（仅管理员）

- 子菜单：用户管理 / 角色权限 / 设备型号库 / 流程模板 / 知识库健康度 / 系统日志
- 每个子页为标准 CRUD 表格 + 详情抽屉

## 五、交互与微动效规则

| 场景     | 规则                                                                         |
| -------- | ---------------------------------------------------------------------------- |
| 加载     | 骨架屏优先，避免整页 spinner；AI 输出用打字机流式                            |

## 八、移动端扩展设计要求 ⭐（重要）

> **背景**：车间现场一线检修员经常**单手、戴手套、在强光/弱光/油污环境下**使用手机或平板登录系统完成检索与作业指引。PC 端的三栏布局在移动端不可用，必须做"移动优先"的差异化设计。

### 8.1 移动端断点与设备策略

| 断点       | 设备              | 策略                              |
| ---------- | ----------------- | --------------------------------- |
| < 640px    | 手机竖屏          | **移动端专属布局**，底部 Tab 导航 |
| 640–1024px | 手机横屏 / 小平板 | **平板适配布局**，可侧边抽屉      |
| ≥ 1024px   | 桌面 / 大平板     | PC 端三栏布局                     |

**建议技术方案**：用 Vant / NutUI / Ant Design Mobile / shadcn + Tailwind Responsive 实现；同一套代码用 Media Query + 容器查询切换布局。

### 8.2 移动端全局规范

- **屏幕适配**：基于 375px 设计稿，rem 适配；安全区适配 iPhone 刘海/灵动岛（`env(safe-area-inset-*)`）
- **触控目标**：所有可点击元素 ≥ **44×44px**（苹果 HIG 标准），按钮高度 ≥ 48px，间距 ≥ 8px
- **字体**：基础字号 16px（防止 iOS 自动放大），标题 18–22px，描述 14–15px
- **配色**：与 PC 端保持一致，但**橙色按钮饱和度提升 10%**（户外强光下更醒目）
- **暗色模式**：默认跟随系统；强光下提供"高对比度模式"（背景纯白 + 文字纯黑 + 按钮加粗）
- **离线能力**：核心检索流程支持离线缓存最近 50 条记录 + 离线 SOP 模板
- **手套模式**（加分项）：设置中可开启，所有可点击区域放大 1.3 倍

### 8.3 移动端导航结构

**顶部 AppBar（高 48px）**：

- 左：汉堡菜单（唤起侧抽屉）
- 中：当前页面标题
- 右：搜索图标 + 通知铃铛

**底部 Tab 栏（高 56px + 安全区）**——只放 4 个最高频入口：

1. **首页** 🏠（工作台）
2. **检索** 🔍（多模态检索）
3. **指引** 📋（当前正在进行的作业指引，如果有则高亮带数字徽标）
4. **我的** 👤（个人中心 + 历史/收藏入口）

**侧抽屉菜单**（从左滑入，宽 80% 屏宽）：

- 用户头像 + 姓名 + 角色徽标
- 完整菜单：工作台 / 多模态检索 / 作业指引 / 知识库 / 案例审核 / 知识图谱 / 系统管理
- 底部：设置 / 切换深色模式 / 退出登录

#### 8.4.3 多模态检索页（移动）⭐ 重点优化
