## 项目背景

本项目是第十五届中国软件杯 A1 赛题——「基于多模态大模型技术的设备检修知识检索与作业系统」，前端使用 Vue 3 + TypeScript + Vite，后端是 FastAPI（运行在 `http://127.0.0.1:8000`）。

上一轮已完成 12 个接口对接（auth/kb/chat/ticket），Vite 代理已配好，但实测发现以下 7 个问题需要修复。

后端接口说明（来自 `NEEDS/后端环境搭建及接口文档.md`）：

- 后端是 FastAPI，JWT 鉴权，登录返回 `access_token`
- 大部分接口需要在 Header 中携带 `Authorization: Bearer <token>`
- 后端直接返回 JSON 体，不裹 `{code, msg, data}`

## 问题 1：所有接口返回 401 Unauthorized / 404 Not Found

### 现象

```
POST /api/auth/login     → 401 Unauthorized
POST /api/chat/query     → 401 Unauthorized
GET  /api/ticket         → 401 Unauthorized
GET  /api/kb/list        → 401 Unauthorized
GET  /api/kg/graph       → 404 Not Found
```

### 根因分析

1. **登录接口本身返回 401**——说明前端发送的登录请求体格式与后端预期不匹配（FastAPI 常见的是 OAuth2PasswordRequestForm，需要 `application/x-www-form-urlencoded` 格式的 `username` + `password`，而前端可能发的是 JSON `{username, password}`）。
2. **其他接口 401**——因为登录失败拿不到 token，后续请求自然无授权。
3. **`/api/kg/graph` 404**——后端没有这个接口，前端不应该调用它，或需要做降级处理。

### 修复要求

#### 1.1 修复登录请求格式

打开 `src/api/auth.ts`，检查 `login()` 函数：

- 如果后端用的是 `OAuth2PasswordRequestForm`，请求体必须是 `URLSearchParams` 格式：

  ```typescript
  const body = new URLSearchParams();
  body.append("username", username);
  body.append("password", password);

  return rawCall<LoginResult>("/api/auth/login", {
    method: "POST",
    headers: { "Content-Type": "application/x-www-form-urlencoded" },
    body: body.toString(),
  });
  ```

- 如果后端接受 JSON，则保持 JSON 格式但确认字段名一致。

**判断方法**：先查看 `NEEDS/后端环境搭建及接口文档.md` 中 `/api/auth/login` 的请求格式要求，或用 curl 测试：

```bash
# 测试 form 格式
curl -X POST http://127.0.0.1:8000/api/auth/login -d "username=admin&password=123456"

# 测试 JSON 格式
curl -X POST http://127.0.0.1:8000/api/auth/login -H "Content-Type: application/json" -d '{"username":"admin","password":"123456"}'
```

同样检查 `register()` 的请求格式。

#### 1.2 确保 token 正确存储与携带

- 登录成功后，确认 `access_token` 被存到 `localStorage`（或 `pinia store`）。
- 在 `src/api/request.ts` 中，确认所有需要鉴权的请求都自动附带 `Authorization: Bearer <token>` header。
- 检查 `rawCall` 和 `safeCall` 的拦截器是否正确注入 token。

#### 1.3 处理 `/api/kg/graph` 404

- 如果后端暂无知识图谱接口，前端不应主动请求它。
- 找到调用 `/api/kg/graph` 的地方，加上 try-catch 或条件判断，404 时静默降级（不显示图谱模块，或显示"暂无数据"占位）。
- **不要移除图谱功能的代码结构**，只是做降级展示。

#### 1.4 注册接口修复

- 检查 `src/api/auth.ts` 中 `register()` 的请求格式是否与后端 `/api/auth/register` 一致。
- 确认注册成功后能自动跳转到登录页。

## 问题 2：前端存在大量假数据，不是用户自主上传的数据

### 现象

知识库列表、工单列表、SOP 列表等页面显示的是硬编码的 mock 数据，而非后端真实数据。

### 修复要求

#### 2.1 优先使用真实接口数据

在以下模块中，**当后端可用时必须展示真实数据**，mock 仅作为后端不可用时的兜底：

| 模块       | 真实接口               | 当前状态         |
| ---------- | ---------------------- | ---------------- |
| 知识库列表 | `GET /api/kb/list`     | 应替换 mock 列表 |
| 工单列表   | `GET /api/ticket`      | 应替换 mock 工单 |
| 检索结果   | `POST /api/chat/query` | 应替换 mock 搜索 |
| 作业指引   | `GET /api/ticket/{id}` | 应替换 mock SOP  |

**规则**：

- 接口请求成功 → 用真实数据渲染
- 接口请求失败（网络错误/超时/500）→ 降级到 mock 数据，并在 UI 上给出提示"当前为离线模式，展示示例数据"
- **绝不允许**：接口返回空数组 `[]` 时用 mock 填充——空就是空，代表用户确实没有数据

#### 2.2 清理无意义的硬编码数据

- 搜索页面的"热门搜索""历史记录"如果是硬编码的，改为从 `localStorage` 读取用户真实历史。
- 工单列表的 mock 数据集中，删掉明显虚构的测试条目，只保留 1-2 条标注为"示例"的兜底数据。

#### 2.3 workflow.ts 的 mock 合并逻辑

当前 `listFlows()` 拼接了真实工单 + mock SOP。修改为：

- 后端有工单 → 只展示真实工单
- 后端无工单 → 显示空状态 + "暂无工单，点击新建 AI 工单"引导
- **不再混合展示 mock 和真实数据**，这会让用户困惑

## 问题 3：登录界面无法注册

### 现象

登录页面没有注册入口，或注册按钮点击无反应。

### 修复要求

#### 3.1 添加注册入口

在登录表单下方增加"没有账号？立即注册"链接，点击后切换到注册表单。

#### 3.2 注册表单字段

注册表单应包含：

- 用户名（username）
- 密码（password）
- 确认密码（confirmPassword）
- 角色（role）——下拉选择：一线检修人员 / 审核人员 / 管理员

#### 3.3 注册流程

1. 前端校验：密码一致性、必填项
2. 调用 `POST /api/auth/register`，请求格式与后端文档对齐
3. 注册成功 → 提示"注册成功，请登录" → 自动跳回登录表单
4. 注册失败 → 显示后端返回的错误信息（如"用户名已存在"）

#### 3.4 登录表单修改

登录表单只需：

- 用户名
- 密码
- 登录按钮

角色选择由后端根据用户信息返回，不再需要前端手动选角色。如果后端确实需要前端传 role，在注册时传即可，登录时不需要。

## 问题 4：管理员账户没有管理上传修改文档的界面

### 现象

管理员登录后，没有专门的管理后台来管理知识库文档（增删改查）。

### 修复要求

#### 4.1 新增管理后台页面

创建 `src/views/admin/KnowledgeAdmin.vue`，仅 `role === 'admin'` 可访问：

功能清单：
| 功能 | 说明 |
|------|------|
| 文档列表 | 调用 `GET /api/kb/list`，展示所有已入库文档（名称、上传时间、chunk 数、状态） |
| 文档上传 | 调用 `POST /api/kb/upload`，支持拖拽上传，支持 PDF/DOCX/TXT/MD |
| 文档删除 | 调用 `DELETE /api/kb/{id}`，二次确认后删除 |
| 文档预览 | 点击文档名称可查看分块内容（如有对应接口） |
| 搜索文档 | 本地过滤文档列表，支持按名称/类型筛选 |

#### 4.2 添加路由

在路由配置中增加：

```typescript
{
  path: '/admin/knowledge',
  name: 'KnowledgeAdmin',
  component: () => import('@/views/admin/KnowledgeAdmin.vue'),
  meta: { requiresAuth: true, roles: ['admin'] }
}
```

#### 4.3 添加导航入口

- 管理员登录后，在侧边栏/导航栏中增加"知识库管理"入口
- 非管理员登录时不显示该入口

#### 4.4 权限守卫

路由守卫中检查角色，非 admin 访问 `/admin/*` 时重定向到首页。

## 问题 5：检索界面不够简洁，应以对话式布局为中心

### 现象

检索页面布局过于复杂，输入框不在视觉中心，不像主流 AI 对话页面那样简洁直观。

### 参考设计

仿照 ChatGPT / Kimi / 豆包 等 AI 对话页面的布局风格：

- 初始状态：页面中央一个大的输入框 + 简短提示语
- 对话状态：上方是历史对话流，下方是固定的输入框
- 整体风格：留白充足、图标精简、视觉焦点在输入区

### 修复要求

#### 5.1 初始状态（无对话时）

```
┌─────────────────────────────────────┐
│                                     │
│                                     │
│         🔍 设备检修智能检索          │
│                                     │
│   ┌─────────────────────────────┐   │
│   │  输入设备故障描述或检修问题... │   │
│   │                    [📎] [➤] │   │
│   └─────────────────────────────┘   │
│                                     │
│   💡 示例：离心泵振动异常如何排查？   │
│   💡 示例：电机过热保护检修流程      │
│                                     │
└─────────────────────────────────────┘
```

- 输入框宽度：PC 端 max-width 720px，移动端 100%
- 支持附件按钮（📎）用于上传设备图片
- 示例提示可点击，点击后自动填入输入框

#### 5.2 对话状态（有历史记录时）

```
┌─────────────────────────────────────┐
│  [用户消息]                         │
│  离心泵振动异常如何排查？            │

└─────────────────────────────────────┘
```

- 输入框固定在底部
- 对话区域可滚动
- AI 回复展示：摘要 + 来源引用（可折叠）
- 移动端同理，适配小屏幕

#### 5.3 视觉规范

- 背景：浅灰或纯白，干净为主
- 输入框：圆角 12px，带微妙阴影
- AI 回复卡片：浅色背景块，左侧带竖线标识
- 来源引用：小字 + 链接样式
- 去掉不必要的装饰性元素和多余按钮

#### 5.4 需要修改的文件

- `src/views/search/Pc.vue` — PC 端检索页重写布局
- `src/views/search/Mobile.vue` — 移动端检索页适配
- `src/views/search/MobileInputBar.vue` — 移动端输入框组件

## 问题 6：首页设计重心偏移——应直接展示检索界面而非工作台

### 现象

应用首页（`/`）展示的是"工作台"概览，而非核心的检索功能。对于设备检修场景，检索是最高频操作，应该是首页。

### 修复要求

#### 6.1 调整路由默认页

将检索页面设为应用根路径 `/` 的默认展示：

```typescript
// router/index.ts
{
  path: '/',
  redirect: '/search'  // 首页直接跳转到检索
}
```

或直接在 `/` 路径下渲染检索组件。

#### 6.2 保留工作台但降级为次级入口

- 工作台移到 `/workspace` 路径
- 导航栏中"工作台"排在"检索"之后
- 侧边栏顺序调整为：**检索** > 工单管理 > 知识库 > 工作台 > （管理员）知识库管理

#### 6.3 移动端 Tab Bar 调整

底部 Tab Bar 第一个 Tab 改为"检索"（🔍），而非"工作台"。

## 问题 7：知识上传过于死板，无法自由添加知识；案例审核不能自由更改知识

### 现象

当前知识上传流程是固定步骤的向导式（4步），不够灵活。审核人员也无法直接修改/编辑已入库的知识内容。

### 修复要求

#### 7.1 知识上传改为灵活模式

**当前**：4 步向导（选择类型 → 填写信息 → 上传文件 → 确认提交）

**改为**：单页面上传 + 可选元数据

```
┌─────────────────────────────────────────────┐
│  添加知识                                    │
│                                             │
│  📤 拖拽文件到此处，或点击选择文件            │
│     支持 PDF / DOCX / TXT / MD              │
│                                             │
│  ┌───────────────────────────────────────┐   │
│  │  已选文件：设备检修手册v3.pdf  [✕]     │   │
│  └───────────────────────────────────────┘   │
│                                             │
│  标签（可选）：[+检修] [+手册] [+自定义]      │
│  备注（可选）：________________               │
│                                             │
│              [上传入库]                      │
│                                             │
│  ─── 已入库文档 ───                          │
│  📄 设备检修手册v2.pdf  12 chunks  [🗑️]     │
│  📄 故障案例集.docx     8 chunks   [🗑️]     │
│  📄 常见问题.txt        3 chunks   [🗑️]     │
└─────────────────────────────────────────────┘
```

- 拖拽上传区域大而明显
- 标签和备注为可选，不强制
- 上传按钮点击后直接调用 `/api/kb/upload`，无需多步确认
- 下方实时展示已入库文档列表（调用 `GET /api/kb/list`）
- 支持多文件同时上传
- 上传过程中显示进度

#### 7.2 案例审核增加编辑能力

对于审核人员（auditor）和管理员（admin）：

- 在知识库文档列表中，每个文档增加"编辑"按钮
- 点击编辑 → 弹出对话框，显示文档的分块内容（chunks），可逐块修改文字
- 修改后调用后端更新接口（如有）；如后端暂无更新接口，先在前端实现 UI，接口调用处标注 `// TODO: 等待后端提供 PUT /api/kb/{id}/chunks 接口`
- 审核人员可标记文档状态：待审核 → 已通过 / 已驳回

#### 7.3 自由添加文本知识

除了文件上传，增加"手动录入"入口：

- 点击"手动录入"按钮 → 弹出文本编辑区域
- 用户可直接粘贴或输入知识文本
- 提交时将文本保存为 .txt 临时文件后调用 `/api/kb/upload`
- 或如果后端支持纯文本上传接口，直接调用

## 修复顺序与验收标准

按以下顺序逐项修复，每项完成后运行 `npx vue-tsc --noEmit && npx vite build` 确认无报错：

| 顺序 | 问题             | 验收标准                                                                            |
| ---- | ---------------- | ----------------------------------------------------------------------------------- |
| 1    | 401/404 接口错误 | 登录可成功获取 token；其他接口正常返回数据；`/api/kg/graph` 404 时 UI 不报错        |
| 2    | 假数据清理       | 后端可用时展示真实数据；后端不可用时降级到 mock 并标注"离线模式"；空数据不填充 mock |
| 3    | 注册功能         | 登录页有注册入口；注册表单完整可用；注册成功跳回登录                                |
| 4    | 管理员界面       | admin 可访问知识库管理页；可查看/上传/删除文档；非 admin 无入口                     |
| 5    | 检索界面重构     | 初始状态居中大输入框；对话状态底部固定输入；视觉简洁类 ChatGPT                      |
| 6    | 首页重心调整     | `/` 直接展示检索页；工作台移至次级路由；导航顺序检索优先                            |
| 7    | 知识管理灵活化   | 上传改为拖拽式单页面；审核可编辑；支持手动录入文本知识                              |

## 注意事项

1. **不要破坏已有的移动端适配**——所有修改需同时检查 PC 和 Mobile 视图
2. **mock 兜底逻辑保留**——用于演示场景，但要确保真实接口优先
3. **类型安全**——所有新增/修改的代码必须通过 `vue-tsc --noEmit`
4. **后端接口格式**——以 `NEEDS/后端环境搭建及接口文档.md` 为准，不确定时先用 curl 测试
5. **权限控制**——admin/auditor/worker 三种角色看到不同功能，前端做 UI 层面的权限隐藏（真正的鉴权在后端）
6. **渐进式修改**——每个问题独立修复，不要一次性大改导致难以回退
