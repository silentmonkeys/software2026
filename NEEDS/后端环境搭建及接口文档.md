## 一、环境搭建

### 1. 克隆仓库

```bash
git clone https://github.com/silentmonkeys/software2026.git
cd software2026
git checkout main
```

### 2. 安装 Python 依赖

```bash
cd backend
pip install -r requirements.txt            # 运行依赖
pip install -r requirements-dev.txt        # 测试依赖：pytest + httpx（可选，跑测试用）
```

> Python 3.10+。建议用 conda 环境 `software`（项目默认）。

### 3. 配置环境变量

```bash
cp .env.example .env
```

打开 `.env`，**至少配置以下三项**：

```ini
DASHSCOPE_API_KEY=sk-你的密钥           # 阿里云百炼申请：https://bailian.console.aliyun.com
JWT_SECRET=必须改！                       # FIX9 C1：默认值会让后端拒绝启动
                                         #   生成：python -c "import secrets; print(secrets.token_hex(32))"
CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173   # 生产改为前端实际域名
# 以下三项一般不用动（相对路径会自动解析为 backend/ 下的绝对路径）
DB_URL=sqlite:///./loongchip.db
CHROMA_DIR=./chroma_db
UPLOAD_DIR=./uploads
DEBUG=false
ALLOW_INSECURE_JWT=false                # 开发/测试想绕过 JWT_SECRET 强制校验时置 true
```

> **FIX9 C1**：`JWT_SECRET` 仍为默认值时后端 `raise SystemExit` **拒绝启动**；`ALLOW_INSECURE_JWT=true` 或 `DEBUG=true` 可临时绕过（仅警告）。
> **FIX9 C3**：`CORS_ORIGINS` 默认仅放行本地前端；逗号分隔多域名；`*` = 完全开放（不推荐生产）。

### 4. 启动后端

```bash
uvicorn app.main:app --reload     # → http://127.0.0.1:8000
```

启动时自动：(a) 建表（SQLite `loongchip.db`），(b) 跑 `run_migrations()` 给旧库补 FIX5/FIX6 列，(c) 播种默认管理员 `admin / 123456`（受 `is_default_admin` 保护，不可删/改名/降权）。

浏览器打开 `http://127.0.0.1:8000/docs` 看 Swagger。

### 5. 跑测试（可选）

```bash
python -m pytest -q tests          # auth / security / health
```

## 二、接口总览

鉴权方式：**JWT Bearer Token**。除 `GET /api/health`、`GET /api/kb/image/{name}`（`<img>` 标签不带 Authorization 头，FIX7）外，所有接口必须带 `Authorization: Bearer <token>`。

> **单点登录（FIX6 第 10 项）**：`User.token_version` 每次登录 / 改密递增并写入 JWT 的 `tv` 字段；`get_current_user` 校验不一致即 `401 "账号已在其他设备登录"`，旧设备 token 立即失效。

### 认证模块 `/api/auth`

| 方法 | 路径 | 说明 | 请求体 |
|---|---|---|---|
| POST | `/api/auth/register` | 注册（**强制写入 `worker`**，忽略 role 字段） | `{"username":"","password":""}` |
| POST | `/api/auth/login` | 登录 → JWT（每次 `token_version` +1） | `{"username":"","password":""}` |
| GET | `/api/auth/me` | 当前用户信息 | — |
| PUT | `/api/auth/change-password` | 修改自己的密码（**必须校验旧密码**） | `{"old_password":"","new_password":""}` |

### 用户管理 `/api/admin`（管理员）

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/admin/users` | 用户列表 |
| POST | `/api/admin/users` | 建号（未填密码 → **随机一次性口令**，仅本次回显；FIX9 C5） |
| PUT | `/api/admin/users/{id}` | 改用户名 / 角色（默认 admin 不可改名/降权） |
| PUT | `/api/admin/users/{id}/role` | 仅改角色 |
| PUT | `/api/admin/users/{id}/reset-password` | 重置为**随机一次性口令** + 旧会话失效（不再 `123456`；FIX9 C5） |
| DELETE | `/api/admin/users/{id}` | 删除（默认 admin / 当前登录账号不可删） |

### 知识库 `/api/kb`

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/kb/upload` | 上传文档（form-data：`file` + 可选 `parent_id` 关联附件；worker→`pending`，auditor/admin→`approved`） |
| POST | `/api/kb/text` | 纯文本经验入库 |
| POST | `/api/kb/text-with-files` | 文本经验 + 支撑附件（先建主条目拿 `parent_id`，再挂附件） |
| GET | `/api/kb/list` | 列表（主条目下挂 `attachments[]`） |
| GET | `/api/kb/{id}` | 文档详情 |
| GET | `/api/kb/{id}/download` | 原始文件下载（PDF/DOCX inline 预览，Blob URL） |
| GET | `/api/kb/{id}/export?format=pdf\|md` | 导出 PDF（reportlab CJK 字体回退）/ Markdown |
| GET | `/api/kb/image/{image_name}` | 解析出的文档图片（引用面板缩略图用，免鉴权） |
| PUT | `/api/kb/{id}` | 编辑文档（审查员/管理员，内容变更且已通过则**重建向量**；FIX9 H1 原子化） |
| POST | `/api/kb/{id}/review` | 审核 通过 / 驳回（级联附件） |
| DELETE | `/api/kb/{id}` | 删除文档 |

### 智能问答 `/api/chat`

| 方法 | 路径 | 说明 | 请求 |
|---|---|---|---|
| POST | `/api/chat/query` | 多模态问答（**SSE 流式**，`text/event-stream`；FIX9 H5） | form-data：`question`（可空，纯图片查询）+ 可选 `image` |
| POST | `/api/chat/correct` | 保存用户对 AI 回答的手动修正 | form-data：`qa_log_id`、`corrected_answer` |

`/api/chat/query` 的 SSE 事件类型：
- `token` —— 增量文本（前端逐字拼接，打字机效果）
- `meta` —— `{qa_log_id, answer, sources[], recommended_tickets[]}`，流末尾给出完整结构
- `error` —— 错误
- `done` —— 结束

`sources[]` 每条含 `doc_id` / `doc_title` / `snippet` / `images[]`（`images` 来自 PDF 解析出的 `[图片文件]` 标记，FIX8）。

### 工单 `/api/ticket` + `/api/workflow`

工单是**平台级共享**资源；每用户独立进度（`user_ticket_progress`），事件写 `ticket_event` 供时间线。

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/ticket` | 创建工单（前端先调 `/recommend`，全拒才开新表单） |
| GET | `/api/ticket` | 列表（我的 + 推荐；软删除的进度落回推荐桶，FIX6 第 3 项） |
| POST | `/api/ticket/recommend` | 相似工单推荐 |
| GET | `/api/ticket/history` | 个人历史（含已删除，带理由） |
| POST | `/api/ticket/{id}/add` | 加入他人工单（新建独立进度） |
| GET | `/api/ticket/{id}` | 工单详情 |
| PATCH | `/api/ticket/{id}/progress` | 推进步骤（传 `stepDone` 数组；事件携带 `stepIndex` 供时间线渲染，FIX6 第 4 项） |
| PATCH | `/api/ticket/{id}` | 兼容旧前端：标记我的进度完成 |
| DELETE | `/api/ticket/{id}` | 删除（已完成默认理由"已完成"；未完成**必须填理由**：需求错误/误触/其他） |
| GET | `/api/ticket/{id}/timeline` | 时间线（员工只看自己，审查员/管理员看全部） |
| GET | `/api/ticket/{id}/tools` · `/manuals` | 工单关联工具 / 手册 |
| GET | `/api/workflow/{id}/tools` · `/manuals` | 同上（`/api/workflow` 别名路由） |

### 知识图谱 `/api/kg`

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/kg/graph` | 图谱节点 + 边（支持 `filter_doc_id`；每节点带 `source_docs[]`，FIX6 第 8 项） |
| PUT / DELETE | `/api/kg/node/{node_id}` | 节点编辑 / 删除（审查员/管理员，落 `kg_overrides`） |
| PUT / DELETE | `/api/kg/edge/{edge_id}` | 边编辑 / 删除（`edge_id` = `"{minSource}|{maxTarget}"` 排序） |
| GET | `/api/kb/{doc_id}/chunk/{chunk_id}` | 节点对应原文片段 |

### 健康检查

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/health` | **深度探活**（FIX9 H3）：`{ok, app, db, chroma, dashscope}`，任一依赖异常 `ok=false` |

## 三、前端调用示例

前端实际封装见 `src/api/request.ts`：`safeCall(fn)` 处理 `{code,msg,data}` 信封、`rawCall(fn)` 处理直接返回 JSON 的接口，二者失败均**抛错**（无 mock 兜底，FIX5 第 16 项）。Token 按路由前缀隔离存储（FIX6 第 11 项）：`#/admin*` 用 `localStorage.admin_token`，其余用 `user_token`。

### 登录

```js
import { safeCall } from '@/api/request'
import { rawCall } from '@/api/request'
import api from '@/api/request'   // axios 实例

// 登录（rawCall：直接返回 JSON）
const { access_token, role } = await rawCall(() =>
  api.post('/api/auth/login', { username: 'admin', password: '123456' })
)
// 写 token 必须走封装，按当前路由前缀落到 admin_token / user_token
import { writeActiveToken } from '@/api/request'
writeActiveToken(access_token)
```

### 知识库

```js
// 上传 PDF（form-data）
const form = new FormData()
form.append('file', file)
await safeCall(() => api.post('/api/kb/upload', form))   // → { doc_id, chunks, status, parent_id }

// 列表（safeCall：解 data 信封）
const list = await safeCall(() => api.get('/api/kb/list'))

// 下载 / 预览（拿 Blob URL）
const blob = await api.get(`/api/kb/${id}/download`, { responseType: 'blob' })
const url = URL.createObjectURL(blob.data)   // 用完记得 revokeObjectURL（FIX9 H16）
```

### 问答（SSE 流式）

`/api/chat/query` 是 SSE，**不能用 axios**（EventSource 不支持 POST/multipart），前端用 `fetch` + `ReadableStream` 手动解析，封装在 `src/api/search.ts` 的 `multimodalSearchStream()`：

```js
import { multimodalSearchStream } from '@/api/search'

const result = await multimodalSearchStream(
  { question: '压缩机异响怎么排查', images: [] },
  {
    onToken: (acc) => updateMessage(acc),   // 逐 token 渲染打字机
  }
)
// result.answer / result.sources / result.recommendedTickets
```

### 工单

```js
// 创建（前端会先调 /api/ticket/recommend）
const ticket = await safeCall(() =>
  api.post('/api/ticket', { device: '空压机', fault: '异响' })
)

// 推进步骤（stepDone = 已完成步骤 id 数组）
await safeCall(() =>
  api.patch(`/api/ticket/${id}/progress`, { stepDone: ['pre-check'] })
)

// 时间线
const events = await safeCall(() => api.get(`/api/ticket/${id}/timeline`))
```

## 四、常见问题

### 后端启动直接退出（SystemExit）
`JWT_SECRET` 仍是默认值（FIX9 C1）。在 `.env` 设强随机 `JWT_SECRET`，或临时 `ALLOW_INSECURE_JWT=true`。

### 401 Unauthorized
未登录 / token 过期 / **被其他设备挤下线**。若 `detail` 含"其他设备"，是单点登录触发（`token_version` 不一致），需重新登录。

### 422 Validation Error
请求体格式不对。对照上面各接口的请求体。

### `/api/chat/query` 拿不到 JSON
它是 SSE 流（`text/event-stream`），不是普通 JSON 响应（FIX9 H5）。前端用 `multimodalSearchStream` 解析；curl 调试加 `-N`：`curl -N -H "Authorization: Bearer <token>" -F question=... http://127.0.0.1:8000/api/chat/query`。

### 500 Internal Server Error
后端报错。FIX9 后 LLM / 检索失败会转 `502` 并返回简要信息（不再泄露堆栈）；看 `uvicorn` 窗口的 traceback 定位。
